# LeetCode #20 Valid Parentheses

今日题目：LeetCode #20 Valid Parentheses

题意概述：
给定一个只包含 `(`、`)`、`[`、`]`、`{`、`}` 的字符串，判断括号是否有效。有效的定义是：
1. 左括号必须用相同类型的右括号闭合；
2. 左括号必须按正确顺序闭合；
3. 每个右括号都必须有对应的最近未匹配左括号。

---

## 解法一：暴力反复删除配对括号

### 思路要点
最直观的办法：如果字符串里存在 `()`、`[]`、`{}`，就把它们删掉；不断重复，直到字符串不再变化。
最后如果删成空串，说明有效；否则无效。

这类做法能做，但效率一般，属于“模拟消消乐”。

### 时间/空间复杂度
- 时间复杂度：`O(n^2)` 到 `O(n^3)`，取决于字符串替换实现；整体不优
- 空间复杂度：`O(n)`

### 关键边界 / 易错点
- 空串应视为有效
- 长度为奇数时一定无效
- 替换时要持续循环，直到字符串不再变化
- 大输入下性能较差

### Python 代码
```python
class Solution:
    def isValid(self, s: str) -> bool:
        if len(s) % 2 == 1:
            return False

        prev = None
        while prev != s:
            prev = s
            s = s.replace('()', '').replace('[]', '').replace('{}', '')

        return s == ''
```

## 解法二：递归按相邻可消对不断收缩

### 思路要点
和解法一同思路，但不用批量 `replace`，而是每次扫描字符串，找到一个相邻合法对就删掉，然后递归处理剩余字符串。

这本质上仍是暴力缩串，只是写法更“显式”。

### 时间/空间复杂度
- 时间复杂度：`O(n^2)`
- 空间复杂度：`O(n^2)`，递归切片会产生额外开销

### 关键边界 / 易错点
- 没找到任何一对可消括号时，应立即判失败
- 递归基：空串为有效
- Python 频繁切片会有额外性能损耗

### Python 代码
```python
class Solution:
    def isValid(self, s: str) -> bool:
        pairs = {'()', '[]', '{}'}

        def dfs(cur: str) -> bool:
            if cur == '':
                return True
            if len(cur) % 2 == 1:
                return False

            for i in range(len(cur) - 1):
                if cur[i:i+2] in pairs:
                    return dfs(cur[:i] + cur[i+2:])
            return False

        return dfs(s)
```

## 解法三：数组模拟栈，右括号时回看栈顶

### 思路要点
有效括号的核心约束其实是“后开的先关”，天然适合栈。

做法：
- 遇到左括号，入栈
- 遇到右括号，检查栈顶是否为对应左括号
  - 是：弹栈
  - 否：直接返回 `False`
- 最后栈为空则有效

这是这题最主流的标准解。

### 时间/空间复杂度
- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

### 关键边界 / 易错点
- 一开始就遇到右括号，栈为空，直接无效
- 匹配时必须比较“最近未匹配左括号”，不是任意一个左括号
- 遍历结束后如果栈不空，也无效
- 空串有效

### Python 代码
```python
class Solution:
    def isValid(self, s: str) -> bool:
        stack = []
        mapping = {
            ')': '(',
            ']': '[',
            '}': '{'
        }

        for ch in s:
            if ch in '([{':
                stack.append(ch)
            else:
                if not stack or stack[-1] != mapping[ch]:
                    return False
                stack.pop()

        return len(stack) == 0
```

## 解法四：入栈“期待的右括号”

### 思路要点
这是栈解法的一个更干净变体。

遇到左括号时，不把左括号本身压栈，而是直接把“未来期望匹配的右括号”压栈：
- 遇到 `(`，压 `)`
- 遇到 `[`，压 `]`
- 遇到 `{`，压 `}`

这样遇到右括号时，只需要判断它是否等于栈顶期望值。

这个写法判断分支更少，代码通常更短。

### 时间/空间复杂度
- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

### 关键边界 / 易错点
- 遇到右括号时必须先判断栈是否为空
- 栈顶存的是“预期右括号”，不是左括号
- 最终仍要检查栈是否清空

### Python 代码
```python
class Solution:
    def isValid(self, s: str) -> bool:
        stack = []
        expected = {
            '(': ')',
            '[': ']',
            '{': '}'
        }

        for ch in s:
            if ch in expected:
                stack.append(expected[ch])
            else:
                if not stack or stack[-1] != ch:
                    return False
                stack.pop()

        return not stack
```

## 解法五：最优工程写法——栈 + 奇偶剪枝 + 提前失败

### 思路要点
算法本质仍然是栈，复杂度上已经最优，不可能优于 `O(n)` 扫描。

这里做工程化优化：
1. 长度为奇数直接返回 `False`
2. 用栈保存期待的右括号
3. 一旦不匹配立即失败
4. 结束时检查是否完全配平

这版最适合面试和实战提交：逻辑稳定、分支少、可读性高。

### 时间/空间复杂度
- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

### 关键边界 / 易错点
- `s = ""` 应返回 `True`
- `s = ")("` 这种首字符非法要立即失败
- `s = "((()))"` 合法，但 `s = "(()"` 不合法
- 如果题目输入可能包含非括号字符，需要额外定义行为；LeetCode 20 默认只有括号字符

### Python 代码
```python
class Solution:
    def isValid(self, s: str) -> bool:
        if len(s) % 2 == 1:
            return False

        stack = []
        pair = {
            '(': ')',
            '[': ']',
            '{': '}'
        }

        for ch in s:
            if ch in pair:
                stack.append(pair[ch])
            else:
                if not stack:
                    return False
                expected = stack.pop()
                if ch != expected:
                    return False

        return not stack
```

## 同类题目练习

这题属于**栈 + 字符串匹配**，虽然不是滑窗/链表/前缀和，但和“括号合法性、表达式求值、单调结构”这一类题强相关。可优先练下面几道：

1. **32 / Longest Valid Parentheses**
   - 为什么推荐练：同样是括号题，但从“判合法”升级到“求最长合法子串”，能加深你对括号配对结构的理解。

2. **22 / Generate Parentheses**
   - 为什么推荐练：从“验证结果”反过来做“构造所有合法结果”，能强化你对括号合法状态空间的理解。

3. **71 / Simplify Path**
   - 为什么推荐练：本质也是栈模拟，练的是“遇到一类 token 入栈、遇到另一类 token 出栈”的标准套路。

4. **150 / Evaluate Reverse Polish Notation**
   - 为什么推荐练：也是典型栈题，但对象从括号变成运算数/运算符，适合迁移栈思维。

5. **155 / Min Stack**
   - 为什么推荐练：帮助你把“会用栈”升级到“会设计栈结构”，属于基础数据结构强化题。

6. **394 / Decode String**
   - 为什么推荐练：字符串解析 + 栈，嵌套结构明显，和括号嵌套处理有很强的思维共性。
