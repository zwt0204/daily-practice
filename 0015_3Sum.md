今日题目：LeetCode #15 3Sum

这题目标很明确：找出数组中所有和为 0 的不重复三元组。难点不在“能不能做出来”，而在两件事：
1. 去重是否彻底；
2. 指针移动时会不会漏解或产生重复解。

下面按“从暴力到最优”的顺序给 5 个解法。

解法 1：三重循环暴力枚举
思路要点：
- 枚举 i、j、k 三个位置，检查 nums[i] + nums[j] + nums[k] 是否为 0。
- 为了避免重复三元组，把每个命中的三元组排序后放进 set。
- 这是最直观的写法，适合理解题意，不适合正式面试提交。

时间复杂度：O(n^3)
空间复杂度：O(m)，m 为结果集合大小（以及 set 去重开销）
关键边界 / 易错点：
- 同一个值可能出现多次，不能只按下标不同就直接加入答案。
- 三元组必须做值层面的去重。
- 输入长度小于 3 时要直接返回空列表。

Python 代码：
```python
from typing import List

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        ans = set()
        for i in range(n):
            for j in range(i + 1, n):
                for k in range(j + 1, n):
                    if nums[i] + nums[j] + nums[k] == 0:
                        triple = tuple(sorted((nums[i], nums[j], nums[k])))
                        ans.add(triple)
        return [list(t) for t in ans]
```

解法 2：双重循环 + 哈希查补数
思路要点：
- 固定前两个数中的第一个数 i。
- 然后在 i 后面的区间里做 Two Sum：枚举 nums[j]，查是否存在 target = -(nums[i] + nums[j])。
- 为避免重复，同样把三元组排序后丢进 set。
- 它本质上是“3Sum 降成多次 2Sum”。

时间复杂度：O(n^2)
空间复杂度：O(n)
关键边界 / 易错点：
- seen 集合是每个 i 独立的，不能跨 i 复用。
- 去重不能只靠 seen，结果仍然可能重复，所以通常还要结果 set。
- 若先查再放、还是先放再查，逻辑要一致，否则可能把自己配对进去。

Python 代码：
```python
from typing import List

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        n = len(nums)
        ans = set()

        for i in range(n):
            seen = set()
            for j in range(i + 1, n):
                need = -(nums[i] + nums[j])
                if need in seen:
                    triple = tuple(sorted((nums[i], nums[j], need)))
                    ans.add(triple)
                seen.add(nums[j])

        return [list(t) for t in ans]
```

解法 3：排序 + 二分查找第三个数
思路要点：
- 先排序。
- 固定 i、j 后，第三个数就变成查找 target = -(nums[i] + nums[j])。
- 因为数组有序，可以在 j 右侧做二分。
- 去重方式：跳过重复的 i、j；找到答案后按值加入。

时间复杂度：O(n^2 log n)
空间复杂度：O(1) 或 O(log n)（取决于排序实现栈空间）
关键边界 / 易错点：
- 二分的搜索区间必须是 j + 1 到 n - 1。
- i、j 都要跳过重复值，否则结果会重复。
- 虽然比暴力好，但仍不如双指针版本自然。

Python 代码：
```python
from bisect import bisect_left
from typing import List

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        n = len(nums)
        ans = []

        for i in range(n - 2):
            if i > 0 and nums[i] == nums[i - 1]:
                continue
            for j in range(i + 1, n - 1):
                if j > i + 1 and nums[j] == nums[j - 1]:
                    continue
                target = -(nums[i] + nums[j])
                k = bisect_left(nums, target, j + 1)
                if k < n and nums[k] == target:
                    ans.append([nums[i], nums[j], nums[k]])
        return ans
```

解法 4：排序 + 双指针（标准做法）
思路要点：
- 先排序。
- 固定 nums[i] 后，在右侧区间用左右指针 l、r 寻找两数之和等于 -nums[i]。
- 如果三数和 < 0，说明需要更大值，l 右移；如果 > 0，说明需要更小值，r 左移。
- 找到一个答案后，同时跳过 l 和 r 的重复值。
- 这是这题最经典、最推荐掌握的写法。

时间复杂度：O(n^2)
空间复杂度：O(1) 或 O(log n)
关键边界 / 易错点：
- 排序后若 nums[i] > 0，可直接结束，后面不可能再凑出 0。
- i 要去重：若 nums[i] == nums[i-1]，跳过。
- 命中答案后，l、r 都要越过重复值，否则会重复收集。
- while l < r 的循环条件不要写错。

Python 代码：
```python
from typing import List

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        n = len(nums)
        ans = []

        for i in range(n - 2):
            if nums[i] > 0:
                break
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            l, r = i + 1, n - 1
            while l < r:
                s = nums[i] + nums[l] + nums[r]
                if s < 0:
                    l += 1
                elif s > 0:
                    r -= 1
                else:
                    ans.append([nums[i], nums[l], nums[r]])
                    l += 1
                    r -= 1
                    while l < r and nums[l] == nums[l - 1]:
                        l += 1
                    while l < r and nums[r] == nums[r + 1]:
                        r -= 1

        return ans
```

解法 5：排序 + 双指针 + 剪枝优化（最优工程版）
思路要点：
- 核心仍是排序 + 双指针。
- 在标准做法基础上增加上下界剪枝：
  - 若当前最小可能和都大于 0，后面无需再看，直接 break。
  - 若当前最大可能和都小于 0，说明当前 i 不可能成功，直接 continue。
- 这种优化在数据分布偏极端时更有效，属于“最优但不花哨”的工程写法。

时间复杂度：最坏 O(n^2)
空间复杂度：O(1) 或 O(log n)
关键边界 / 易错点：
- 剪枝条件别写反。
- 命中答案后仍要做双边去重。
- 剪枝是锦上添花，不要因为剪枝把主流程写乱。

Python 代码：
```python
from typing import List

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        n = len(nums)
        ans = []

        for i in range(n - 2):
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            # 最小三数和已经大于 0，后面只会更大
            if nums[i] + nums[i + 1] + nums[i + 2] > 0:
                break

            # 最大三数和仍小于 0，当前 i 不可能组成答案
            if nums[i] + nums[n - 2] + nums[n - 1] < 0:
                continue

            l, r = i + 1, n - 1
            while l < r:
                s = nums[i] + nums[l] + nums[r]
                if s == 0:
                    ans.append([nums[i], nums[l], nums[r]])
                    l += 1
                    r -= 1
                    while l < r and nums[l] == nums[l - 1]:
                        l += 1
                    while l < r and nums[r] == nums[r + 1]:
                        r -= 1
                elif s < 0:
                    l += 1
                else:
                    r -= 1

        return ans
```
